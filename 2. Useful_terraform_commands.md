**Terraform Commands & Their Explanation**

**1. `terraform init`**
Initializes the Terraform working directory.
Downloads required providers (AWS, Azure, etc.) and modules.

    terraform init
**When to use:**
✔ First time you create the project
✔ After adding/changing providers
✔ After using modules

**2. `terraform validate`**
Checks whether your Terraform code syntax is valid.
  terraform validate

**When to use:**
✔ Before running plan or apply
✔ After editing files

**3. `terraform fmt`**
Fixes formatting of `.tf` files. Makes code clean.

```bash
terraform fmt
```

**When to use:**
✔ Before committing to Git
✔ After writing long code


**4. `terraform plan`**
Shows what changes Terraform **will make** without applying them.

```bash
terraform plan
```

**When to use:**
✔ Before apply
✔ To review changes

**5. `terraform apply`**
Applies the plan → creates/modifies/destroys infrastructure.

```bash
terraform apply
```

To skip confirmation:

```bash
terraform apply -auto-approve
```

**6. `terraform destroy`**
Destroys all infrastructure created by Terraform.

```bash
terraform destroy
```

Skip prompt:

```bash
terraform destroy -auto-approve
```

**7. `terraform show`**
Displays the current state or plan in readable format.

```bash
terraform show
```

**8. `terraform state`**
Manages the Terraform **state file**.
Examples:

**View all resources in state:**

```bash
terraform state list
```

**Remove a resource from state:**

```bash
terraform state rm aws_instance.web
```

**Show a specific resource:**

```bash
terraform state show aws_instance.web
```

**9. `terraform output`**
Shows output variables from Terraform.

```bash
terraform output
```

**10. `terraform providers`**
Shows which providers are being used.

```bash
terraform providers
```

**11. `terraform refresh`**
(Deprecated, but good to know)

Updates state file with actual real-time cloud status.

**12. `terraform import`**
Imports existing AWS/Azure resources into Terraform.

```bash
terraform import aws_instance.web i-0123456789abcd
```

**13. `terraform graph`**
Shows dependency graph (used mostly with visualization tools).

```bash
terraform graph
```

**14. `terraform workspace`**
Used to manage multiple environments: dev / qa / prod.

**Create workspace:**

```bash
terraform workspace new dev
```

**Switch workspace:**

```bash
terraform workspace select dev
```

**List all workspaces:**

```bash
terraform workspace list
```

**15. `terraform login`**
Login to Terraform Cloud.

```bash
terraform login
```

**16. `terraform version`**
Shows your installed Terraform version.

```bash
terraform version
```
